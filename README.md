# 🚀 Code With TS

A beginner-friendly repository to learn, explore, and understand **TypeScript** from the ground up.

TypeScript is not a replacement for JavaScript—it's a **superset of JavaScript** that adds static typing, better tooling, and improved developer experience while compiling down to plain JavaScript.

---

## 📖 What is TypeScript?

TypeScript (TS) is an open-source programming language developed by :contentReference[oaicite:0]{index=0}.

It extends JavaScript by introducing:

- Static typing
- Interfaces
- Enums
- Generics
- Type inference
- Better IDE support
- Enhanced code maintainability

Since browsers cannot directly execute TypeScript, the TypeScript Compiler (`tsc`) converts TypeScript code into JavaScript.

---

# 🏗️ TypeScript Compilation Pipeline

When you write TypeScript code and run the compiler, it goes through several internal phases before generating JavaScript.

```text
TypeScript Code
       │
       ▼
    Lexer
       │
       ▼
    Parser
       │
       ▼
    Binder
       │
       ▼
    Checker
       │
       ▼
 Emitter / Generator
       │
       ▼
 JavaScript + .d.ts + Source Maps
```

---

## 1️⃣ TypeScript Code

This is the code written by the developer.

Example:

```ts
function greet(name: string): string {
  return `Hello ${name}`;
}

console.log(greet("Priyanshu"));
```

The compiler starts processing this source code.

---

## 2️⃣ Lexer (Lexical Analysis)

The Lexer breaks the source code into smaller pieces called **tokens**.

Tokens are the smallest meaningful units of a program.

Example:

```ts
let age = 20;
```

Tokens:

```text
let
age
=
20
;
```

### Responsibilities

- Reads raw source code
- Identifies keywords
- Identifies operators
- Identifies literals
- Generates tokens

### Output

```text
Token Stream
```

---

## 3️⃣ Parser (Syntax Analysis)

The Parser takes the token stream generated by the Lexer and builds an **Abstract Syntax Tree (AST)**.

An AST represents the structure of the code.

Example:

```ts
let age = 20;
```

Simplified AST:

```text
VariableDeclaration
├── Identifier(age)
└── NumericLiteral(20)
```

### Responsibilities

- Validates syntax
- Creates AST
- Detects syntax errors

Example Error:

```ts
let = age 20;
```

```text
Syntax Error
```

### Output

```text
Abstract Syntax Tree (AST)
```

---

## 4️⃣ Binder

The Binder walks through the AST and creates relationships between declarations.

It builds a **Symbol Table**.

Example:

```ts
let age = 20;
```

The binder records:

```text
age → Variable Symbol
```

### Responsibilities

- Creates symbols
- Tracks declarations
- Maintains scopes
- Builds symbol tables

### Why is it needed?

Without the Binder, TypeScript wouldn't know:

```ts
let age = 20;

console.log(age);
```

whether `age` refers to a valid variable or not.

### Output

```text
AST + Symbol Table
```

---

## 5️⃣ Checker (Type Checking)

The Checker is the most powerful part of TypeScript.

It uses the AST and Symbol Table to perform type analysis.

Example:

```ts
let age: number = "20";
```

Error:

```text
Type 'string' is not assignable to type 'number'
```

### Responsibilities

- Type checking
- Type inference
- Generic validation
- Interface validation
- Compatibility checks
- Error reporting

Example:

```ts
let age = 20;
```

TypeScript automatically infers:

```ts
age: number
```

This process is called **Type Inference**.

### Output

```text
Type-checked AST
```

---

## 6️⃣ Emitter (Code Generation)

The Emitter converts the TypeScript AST into JavaScript.

Example:

### TypeScript

```ts
const greet = (name: string) => {
  return `Hello ${name}`;
};
```

### Generated JavaScript

```js
const greet = (name) => {
  return `Hello ${name}`;
};
```

Notice that TypeScript-specific type annotations are removed.

### Responsibilities

- Generate JavaScript
- Generate declaration files
- Generate source maps

### Output Files

#### JavaScript

```text
index.js
```

#### Declaration Files

```text
index.d.ts
```

Declaration files expose type information for other projects.

Example:

```ts
declare function greet(name: string): string;
```

#### Source Maps

```text
index.js.map
```

Source maps help developers debug TypeScript code in browsers even though JavaScript is actually running.

# 🎯 Why TypeScript?

### Better Developer Experience

- Auto-completion
- IntelliSense
- Refactoring support

### Type Safety

```ts
function add(a: number, b: number) {
  return a + b;
}
```

Prevents accidental bugs caused by incorrect types.

### Scalability

Perfect for:

- Large applications
- Enterprise software
- React projects
- Node.js backends
- Full-stack development

# 📚 Key Takeaway

The TypeScript compiler is much more than a simple translator.

```text
TypeScript
    ↓
Lexer
    ↓
Parser
    ↓
Binder
    ↓
Checker
    ↓
Emitter
    ↓
JavaScript
```

Each phase has a specific responsibility:

| Phase | Purpose |
|---------|---------|
| Lexer | Converts source code into tokens |
| Parser | Builds the AST |
| Binder | Creates symbols and scopes |
| Checker | Performs type checking |
| Emitter | Generates JavaScript output |

Understanding this pipeline helps developers write better TypeScript code and understand compiler errors more effectively.
